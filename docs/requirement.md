# 南京大学本科新生校规与教务流程 RAG Bot — MVP 需求文档

## 1. 项目背景

南京大学本科新生在入学后普遍面临信息不对称问题：校规分散在多个官网栏目，培养方案和办事流程缺乏统一查询入口。本项目构建一个面向新生的校规与教务流程 RAG（检索增强生成）问答系统，提供带来源引用和风险分级的校规导航服务。

## 2. 目标用户

南京大学本科新生（大一至大二为主要服务对象）。

## 3. 第一版功能范围

### 覆盖主题

- 选课（选课规则、补选退选、容量已满处理）
- 缓考（申请条件、流程、截止时间）
- 补考（资格、报名方式、时间安排）
- 重修（条件、流程、成绩记载）
- 成绩管理（成绩查询、成绩异议、绩点计算）
- 学籍异动（休学、复学、保留学籍、退学）
- 辅修（申请条件、学分要求、课程安排）
- 交换培养（申请、课程认定、学分转换）
- 学业预警（触发条件、处理流程）
- 考试纪律与处分（违纪类型、处分等级、处分期限、解除条件）
- 学生申诉（申诉流程、时效、复查机制）
- 推免研究生（推荐条件、流程、学分绩要求）
- 学分学费（收费标准、补考重修费用、退费办法）
- 奖学金与助学（国家奖学金、人民奖学金、助学金申请条件）
- 常见办事流程（材料提交、部门联系方式）

### 功能能力

1. 用户以自然语言提问。
2. 系统检索知识库，返回带来源引用的回答。
3. 根据问题风险等级采用不同回答策略。
4. 找不到依据时明确拒答。
5. 支持本地命令行、FastAPI 接口两种交互方式。

## 4. 第一版明确不做

- 个人成绩读取或查询
- 个人毕业资格判断
- 自动提交任何申请
- 替代教务员给出个人结论
- 覆盖所有学院全部细则
- 登录系统抓取需认证的页面

## 5. 风险分级

| 等级 | 范围 | 回答策略 |
|------|------|----------|
| low | 查询一般制度、流程入口、文件含义 | 正常回答，引用来源 |
| medium | 缓考、补考、重修、转专业、辅修、交换、成绩等涉及个人情况的问题 | 回答一般规则，加提醒建议咨询学院 |
| high | 退学、处分、作弊、学位、毕业资格、个人学籍状态 | 只解释一般规则，不下个人结论，要求联系学院教务员或相关部门 |

## 6. 高风险问题回复原则

1. 不直接给出个人结论（如"你会被开除""你不能毕业"）。
2. 只解释可查询到的一般规则。
3. 明确建议联系学院教务员或相关部门确认。
4. 回答末尾固定附加风险提示。

## 7. 初步验收标准

- 系统能对 10 个预设 Demo 问题给出回答。
- 前 7 个正常问题有来源引用。
- 后 3 个高风险/无依据问题保守处理或拒答。
- 答案不编造文件名、日期、链接。
- FastAPI `/health` 返回正常。
- FastAPI `/ask` 返回结构化 JSON。
