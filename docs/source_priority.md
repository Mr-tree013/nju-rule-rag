# 资料来源优先级规范

## 1. 优先级定义

| 优先级 | 来源类型 | 说明 | 示例 |
|--------|----------|------|------|
| 1 | 现行校级正式文件 | 本科生院发布、当前有效的校级规章制度 | 《南京大学普通全日制本科生学籍管理规定》 |
| 2 | 本科生院正式通知 | 本科生院官网"通知公告"栏目发布的教务相关通知 | 缓考通知、选课通知、补考安排 |
| 3 | 学院正式通知 | 各学院官网发布的本科教务通知、办事指南 | XX学院本科生选课指南 |
| 4 | 办事指南 | 官方发布的流程说明、办事指引 | 缓考申请流程图、补考报名步骤 |
| 5 | 学生手册 | 官方发布的综合手册，含汇总性信息 | 本科生学生手册 |

## 2. 不进入知识库的内容

**绝对不收录：**

- 群聊经验、论坛帖子、贴吧内容
- 非正式转述（如"听学长说"）
- 没有来源出处的总结或整理
- 个人博客、公众号文章（非官方认证）
- 已废止或过期的文件
- 需要校内 VPN 或登录才能访问的页面（除非获得授权）
- 聊天截图、手写笔记照片

**谨慎收录：**

- 超过 5 年未更新的文件 → 标注"需人工确认时效性"
- 适用范围模糊的文件 → note 标注"适用范围需确认"
- 与现行文件存在矛盾的内容 → 以优先级高的为准，上报人工处理

## 3. 检索排序权重建议

混合检索最终排序时，来源优先级影响：

```
final_score = bm25_norm * 0.45 + vector_norm * 0.45 + priority_bonus * 0.10
```

priority 越低（越权威），`priority_bonus` 越高。

## 4. sources.csv 字段规范

```csv
source_id,title,url,source_type,department,scope,priority,need_login,update_frequency,note
```

| 字段 | 说明 | 示例 |
|------|------|------|
| source_id | 唯一标识，格式: `nju-{部门缩写}-{序号}` | `nju-jw-001` |
| title | 资料标题（中文） | `南京大学普通全日制本科生学籍管理相关文件` |
| url | 公开可访问的 URL | `https://jw.nju.edu.cn/...` |
| source_type | 文件类型 | `html` / `pdf` |
| department | 发布部门 | `本科生院` / `XX学院` |
| scope | 适用范围 | `本科生` / `大一新生` / `全部` |
| priority | 优先级 1-5 | `1` |
| need_login | 是否需要登录查看 | `yes` / `no` |
| update_frequency | 更新频率 | `semester` / `yearly` / `irregular` |
| note | 备注 | 正式文件 / 需人工确认时效 |

## 5. 资料建议来源

1. 南京大学本科生院官网 — 学校文件
2. 本科生院官网 — 通知公告
3. 南京大学教务处网站
4. 1-2 个试点学院的本科教务办事指南页面
5. 南京大学学生手册（公开版本）
6. 学工部/学工处相关规章制度公开页面

## 6. 来源标注规范

回答中引用来源时，统一格式：

```
【依据】
1. 《[文件标题]》— [条款/章节] — [来源URL]
```

如果文件没有公开 URL（如 PDF 上传到知识库），标注：

```
【依据】
1. 《[文件标题]》— [条款/章节]
```
