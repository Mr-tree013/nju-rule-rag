# 风险分级与回答策略

## 1. 风险等级定义

### low（低风险）

一般制度查询、流程入口查询、文件含义解释。

**关键词：**
选课、课程、学分、上课时间、教室、教材、考试时间、假期、校历、宿舍、食堂、图书馆、校园卡、报到、注册、军训、体检

**回答策略：**
正常回答，引用来源文件及条款。可直接给出流程说明和部门指引。

---

### medium（中风险）

涉及个人情况但非处分级别的问题，回答一般规则即可，不应针对个人下结论。

**关键词：**
缓考、补考、重修、成绩、绩点、转专业、辅修、交换、课程认定、学业预警、休学、复学、保留学籍、退课、加课、免修、免听

**回答策略：**
1. 说明一般规则和流程。
2. 加注"具体以个人实际情况和学院通知为准"。
3. 建议联系所在学院教务员确认。
4. `need_human_confirm` 设为 `true`。

---

### high（高风险）

涉及处分、退学、学位、毕业资格等严重后果的问题。

**关键词：**
退学、开除、处分、作弊、违纪、学术不端、学位、毕业资格、毕业不了、学籍取消、严重警告、记过、留校察看、取消学籍、撤销学位、取消考试资格、旷考多次

**回答策略：**
1. 只解释校规中一般性规定。
2. 明确声明"本系统不对个人情况做判断"。
3. 不得给出"你会被开除""你不能毕业"等个人结论。
4. 要求联系学院教务员或相关部门。
5. `need_human_confirm` 设为 `true`。
6. 固定附加高风险提示模板。

---

## 2. 流程类问题判断

**关键词：**
怎么办、在哪里、入口、流程、材料、申请、截止、什么时候、找谁、步骤、网上、系统、提交

如果是流程类问题 + 中等风险主题，回答时应明确标注"以下为一般流程，具体以当学期通知为准"。

---

## 3. 拒答模板

### 无依据可查时

```
我没有在当前知识库中找到足够可靠的依据来回答这个问题。

建议：
1. 咨询所在学院教务员或辅导员。
2. 关注本科生院官网通知公告。
3. 查阅最新版学生手册。
```

### 高风险个人结论类

```
此类问题涉及个人学籍/处分/学位等重大事项，本系统不能也不应直接给出个人结论。

根据校规一般规定：[仅引用可查的一般规则]

建议你尽快联系：
- 所在学院教务办公室
- 本科生院相关部门
- 辅导员或学院学生工作办公室

请以官方书面答复为准。
```

## 4. 风险分类规则（程序实现）

```python
HIGH_RISK_KEYWORDS = [
    "退学", "开除", "处分", "作弊", "学位",
    "毕业不了", "毕业资格", "学籍取消", "严重警告",
    "记过", "留校察看", "取消学籍", "撤销学位",
    "学术不端", "旷考"
]

MEDIUM_RISK_KEYWORDS = [
    "转专业", "辅修", "课程认定", "交换", "缓考",
    "补考", "重修", "学业预警", "绩点", "成绩",
    "休学", "复学", "免修", "免听"
]

PROCESS_KEYWORDS = [
    "怎么办", "在哪里", "入口", "流程", "材料",
    "申请", "截止", "什么时候", "找谁", "步骤",
    "提交", "系统"
]
```

**匹配优先级：** high > medium > low。任一 high 关键词命中即归为 high。

## 5. 人工核验要求

以下情况必须有人工核验才能作为系统最终回答：

1. 高风险问题在 Demo 展示前的所有回答。
2. 使用了 priority=5（学生手册等非正式文件）作为唯一来源的回答。
3. 检索到的 chunk 发布时间超过 3 年。
4. 系统自动判定为"无依据"但评测问题期望有答案的情况。
