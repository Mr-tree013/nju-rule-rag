### 一、适用对象与应用场景

**适用对象**：全校本科生（含交流生、留学生）。文件同时对指导教师和任课教师在毕业论文和教学中的监管职责有延伸规定。

**应用场景**覆盖全过程：
- 日常课程作业、随堂测试、期末论文/报告
- 学科竞赛、创新创业训练计划（国创/省创）
- 本科毕业论文（设计）的开题、研究、撰写、答辩全过程
- 推免、评优等涉及学术成果提交的场景

### 二、禁止代写的核心内容（红线）

| 类别 | 禁止内容 |
|------|---------|
| **论文全篇** | 严禁直接使用 AI 生成毕业论文（设计）的摘要、正文、核心结论或代码 |
| **数据与图表** | 严禁利用 AI 凭空伪造、篡改或润色实验数据、调研结果、统计图表及文献引用 |
| **课程作业** | 严禁直接复制 AI 生成的文本、代码作为日常作业、编程练习或课程小论文提交 |
| **引用列表** | **严禁直接生成学术引用与参考文献列表** |
| **研究假设** | 禁止代写研究（实验）假设提出、选题意义 |
| **方案设计** | 禁止代写方案设计、创新性方法设计 |
| **研究结果** | 禁止代写研究（实验）数据、研究（实验）结果 |
| **分析结论** | 禁止代写分析与讨论、结论总结 |

### 三、允许使用的合规范围 + 申请审批流程

**合规使用范围（须教师审批）：**

| 允许范围 | 说明 |
|---------|------|
| 文献与资料检索 | 寻找研究线索、翻译外文文献、梳理文献大意 |
| 语言润色与纠错 | 对学生自己撰写的文本进行语法纠错、语句通顺度优化（不得改变原意） |
| 代码调试与辅助 | 排查程序 Bug、理解算法逻辑（而非直接生成完整项目） |
| 激发灵感 | 选题阶段的头脑风暴、拓宽研究视角 |

**申请审批流程：**
- **毕业论文阶段**：开题前向指导教师汇报计划使用何种 AI 工具，并在开题报告中明确写明
- **日常课程**：遵循"任课教师负责制"，教师有权决定是否允许。如允许，学生需在提交作业前向教师口头或书面报备
- **核心原则**：须征得任课教师**书面/电子签名同意**

### 四、使用留痕与透明披露要求

原则：**"谁使用、谁披露、谁负责"**

**全过程留痕：**
- 必须保留原始提示词（Prompts）和 AI 生成的原始输出记录
- 形式可以是：截图、对话链接或文本存档
- 供学校随时抽查和学术不端鉴定使用

**透明披露要求（毕业论文等)：**
文末或指定位置必须明确披露——
1. 使用的生成式 AI 工具名称及版本（如 ChatGPT-4o、文心一言 4.0）
2. 使用的具体时间段
3. AI 在成果中发挥的具体作用（如"用于第三章英文文献翻译""用于附录代码 Debug"）

---

### 五、违规处理的分级处分体系

| 行为类型 | 处分 |
|---------|------|
| 日常作业/竞赛违规 | 相关作业计 0 分、取消竞赛资格、学院通报批评或约谈 |
| 毕业论文答辩前发现 | 责令推迟答辩、论文降级、情节严重者取消答辩资格延期毕业 |
| 毕业论文答辩后发现 | 经学术委员会认定学术不端 → 撤销已授予的学士学位、注销学历证书 |
| 纪律处分等级 | 警告 → 严重警告 → 记过 → 留校察看 → **开除学籍**，记入个人档案 |

### 六、AI 使用声明模板

文件附录提供统一模板：**《南京大学本科生生成式人工智能（GAI）工具使用声明表》**

模板包含以下要素（学生签字确认）：

| 要素 | 内容 |
|------|------|
| 学生基本信息 | 姓名、学号、院系、专业 |
| 成果信息 | 课程名称/论文题目、指导教师/任课教师姓名 |
| AI 工具信息 | 工具名称、版本号、开发公司 |
| 使用详情清单 | 表格形式：列出应用在哪些章节/段落、具体用途（数据处理、翻译、润色）、**提示词示例** |
| 诚信承诺 | 学生签字，承诺除声明部分外均为独立完成 |
| 导师/教师意见 | 指导教师或任课教师签字确认 |

### 覆盖的问题确认

> ✅ 课程论文能不能用 ChatGPT？→ "任课教师负责制"，须教师审批
> ✅ AI 润色英文是否算学术不端？→ 语言润色属于合规范围，但须披露
> ✅ 毕设代码用 AI 是否需要声明？→ 是，必须填使用声明表
> ✅ 学校有无统一模板？→ 有，文件附录提供
> ✅ 只用 AI 改语法算违规吗？→ 须教师审批准许，且须披露
> ✅ 小组作业一人用 AI 其他人受影响？→ 需查具体规定；原则上是使用者个人负责
